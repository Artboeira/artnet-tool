# Story 1.2: UI Shell — Dark Theme, shadcn/ui & 3-Tab Navigation

Status: done

## Story

As a developer,
I want shadcn/ui and Tailwind CSS installed with a dark theme and the 3-tab navigation shell (Cue Pad / Monitor / Settings),
so that all UI feature development can build on a consistent, themed component foundation.

## Acceptance Criteria

1. **Given** the application launches, **When** the main window renders, **Then** the UI displays in a dark professional theme with no light-mode flash, **And** the Tailwind CSS dark theme is applied globally.

2. **Given** the main window is open, **When** the user views the navigation, **Then** three tabs are visible: "Cue Pad", "Monitor", and "Settings", **And** clicking each tab switches the active view without page routing.

3. **Given** the shadcn/ui library is installed, **When** any shadcn/ui component (e.g., Button, Tabs) is rendered, **Then** it displays correctly within the dark theme without style overrides needed.

4. **Given** the application window, **When** it is resized to typical desktop dimensions, **Then** the tab layout remains usable and does not overflow or break.

### AppShell Implementation Pattern

```tsx
// src/components/layout/AppShell.tsx
import { useState } from 'react';
import { Tabs, TabsContent, TabsList, TabsTrigger } from '@/components/ui/tabs';

type TabId = 'cue-pad' | 'monitor' | 'settings';

export function AppShell() {
  const [activeTab, setActiveTab] = useState<TabId>('cue-pad');

  return (
    <div className="flex flex-col h-screen bg-background text-foreground overflow-hidden">
      <Tabs
        value={activeTab}
        onValueChange={(v) => setActiveTab(v as TabId)}
        className="flex flex-col flex-1 overflow-hidden"
      >
        <TabsList className="w-full justify-start shrink-0 rounded-none border-b border-border px-2">
          <TabsTrigger value="cue-pad">Cue Pad</TabsTrigger>
          <TabsTrigger value="monitor">Monitor</TabsTrigger>
          <TabsTrigger value="settings">Settings</TabsTrigger>
        </TabsList>

        <TabsContent value="cue-pad" className="flex-1 overflow-auto p-4">
          {/* TODO (Story 3.x): Replace with CuePad feature component */}
          <p className="text-muted-foreground">Cue Pad — coming in Epic 3</p>
        </TabsContent>

        <TabsContent value="monitor" className="flex-1 overflow-auto p-4">
          {/* TODO (Story 2.3): Replace with DmxMonitor feature component */}
          <p className="text-muted-foreground">Monitor — coming in Epic 2</p>
        </TabsContent>

        <TabsContent value="settings" className="flex-1 overflow-auto p-4">
          {/* TODO (Story 2.1+): Replace with Settings feature component */}
          <p className="text-muted-foreground">Settings — coming in Epic 2+</p>
        </TabsContent>
      </Tabs>
    </div>
  );
}
```

### Tab Naming — Discrepancy Notice

The **epics.md ACs** specify: `"Cue Pad"`, `"Monitor"`, `"Settings"` — **follow these**.

The **UX design spec** uses different labels: `"Record"`, `"Perform"`, `"Configure"`. This reflects the UX concept phase terminology and maps to the same tabs:
- Record = Monitor tab
- Perform = Cue Pad tab
- Configure = Settings tab

The architecture.md (`AppShell.tsx` comment) also says "Cue Pad / Monitor / Settings". Implement with the epics.md names. The rename to UX-spec labels (if desired) is a separate decision for a future story.

### Architecture Compliance Rules for This Story

**IPC boundary — always enforced:**
- `AppShell.tsx` does NOT call `invoke()` — it is pure React UI
- `App.tsx` does NOT call `invoke()` — it mounts AppShell only
- All future invoke() calls live exclusively in `src/lib/tauri.ts`

**Component file locations:**
- `src/components/layout/AppShell.tsx` — the app shell (this story)
- `src/components/ui/tabs.tsx` — shadcn generated (do not edit)
- `src/components/ui/button.tsx` — shadcn generated (do not edit)
- Feature placeholder content stays inline in AppShell for now (not in feature files yet — those get populated in their own epics)

**Naming conventions:**
- React component files: `PascalCase` (e.g., `AppShell.tsx`)
- TypeScript types: `PascalCase`, no `I` prefix
- Variables/hooks: `camelCase`

**No routing library:**
- ARTNET-TOOL uses tabbed navigation only — NO react-router, NO TanStack Router
- Tab switching = React `useState`, not URL changes
